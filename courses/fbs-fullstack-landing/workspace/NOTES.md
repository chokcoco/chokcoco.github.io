# 课程准备笔记与生成约束

## 已确认的方向

- 核心不是通用前端课或后端课，而是跨端研发进入 FBS 真实仓库的 Landing 课程。
- 前端与后端工程模块必须强绑定代码仓库，业务作为理解链路和综合实践的上下文。
- 只关注用户点名的 8 个仓库和 `assets/fbs/`，以及这些来源明确指向的 Confluence 页面。
- `fbs-kb` 只作为业务参考来源，不讲授其仓库结构、组织方式或生成机制。
- 所有正式知识章节有效字符不得少于 9000，不设最高限制；写到把学习成果、仓库链路和验证方法讲透为止，不因字符数压缩或注水。
- 2026-07-20 模块五返工新增硬约束：禁止在练习、自检、小结后追加主体知识来补字符；体量不足必须回到原学习路径融合补充。框架、协议、数据库、安全等章节在第一次使用标题术语时，先给零基础定义、架构职责、社区同类方案和当前方案取舍，再进入仓库代码。

## 已发现的关键事实

1. 前端存在三代/三形态工程，不能用一套 React 或 Vue 模板统讲。
2. Vue 与 React 是框架，不是语言。前端语言模块以 JavaScript、TypeScript 和 Web 标准为主，再提供 JSX、Vue SFC 的读码桥梁。
3. 后端三个仓库都是 Go，但 Go 版本不一致：主服务与敏感数据服务为 1.20，税务服务 `go.mod` 为 1.15。
4. `sbs-fbs-server` 同时存在 `app/` 与 `apps/`，课程要解释旧/新模块并存和 Wire 聚合边界。
5. `fbs-sensitive-data-server` 的 module path 仍为 `sbs-fbs-server`，课程只把它作为现状事实和排错边界，不擅自解释历史原因。
6. 前端资料中部分 Node 版本描述与当前 fullstack 文档有差异。正文按各仓库当前 manifest、fullstack 指引和已验证启动线分别说明。
7. `fbs-kb` 中不少主题为 `unknown` 或 `partially_unresolved`。课程中的业务语境只选择有代码校准的部分，不把综合知识库当作绝对真相。

## 已确认的主案例

使用“入库请求/ASN 小型变更”贯穿课程：

- 前端三个仓库都有入库相关页面、路由、API 或共享组件证据。
- `sbs-fbs-server` 有完整 inbound/ASN 的 HTTP、应用、领域、仓储、任务和测试路径。
- `fbs-kb` 的 inbound 主题覆盖 Portal、SC、BR、SEA、CB 多种流程，并明确标记证据状态。
- 后端开发指引直接使用入库接口和 IR 任务作为示例。

为控制范围，实践需求应选一个小型纵向切片，例如“在既有入库列表增加一个可筛选、可校验、可追踪的字段/状态”，不新造大业务流程。

## 已确认项

1. 采用 6 个模块，FBS 业务只穿插在工程章节和最终项目中。
2. 主案例为入库请求/ASN 小型变更。
3. 以研发自学为主，不设单节时长上限，以单章字符限制和完成证据控制深度。
4. - 六个仓库（3 前端 + 3 后端）的 release 分支（2026-07-20）是本轮课程的批准事实基线。
5. 只支持 macOS 和公司开发环境。
6. 最终核心交付为 Markdown 与 HTML，不生成讲师 PPT。
7. 课程仅在公司内部流通，不制作公开脱敏版本；相关内容 URL 可以直接出现。
8. 受众仍按“会本端主语言，另一端为零基础或弱基础”设计。

## 具体操作边界

- 在 test 环境读取数据、启动服务和只读联调可作为正文建议；任何环境写操作仍需按当次任务授权。
- 前后端发版不在课程授课范围，开发交接完成后只用一句话提示遵循团队通用流程，不索取或虚构发布 SOP。

## 希望补充的材料

以下材料会继续提升“独立完成开发与交接”的可信度，按优先级排列：

1. 三个后端仓库的全栈/Onboarding 文档；目前 `sbs-fbs-server` 有 Readme 与后端开发指引，另外两仓缺少同等粒度的 Landing 文档。
2. `fbs-sc-react` 当前分支对应的 fullstack Skill 和 `docs/fullstack.md`；Confluence 有工程梳理，但仓库里未见与另外两前端仓同形态的 fullstack 文档。
3. 一到两个已完成的典型纵向需求材料：PRD、TD、前后端 MR、测试记录和复盘。课程仅内部使用，不要求为公开发布额外脱敏。
4. 前端与后端的当前开发准出标准：最低单测/覆盖率、lint/type-check/build 和接口兼容要求。
5. 可供教学使用的 test 账号、PFB/代理说明和测试数据准备流程。账号与 token 不写入课程正文，应通过安全渠道提供。

## 后续执行建议

用户确认蓝图后，先生成课程首页、术语与环境前置，再按 3-4 章一批生成。建议先做两份标杆章：

- 后端转前端标杆：沿 FBS SC Vue 的路由、Store 与 API 完成一个小改动。
- 前端转后端标杆：沿 Chassis HTTP → Application → Repository 完成一个小接口改动。

标杆章通过人工审核后，再批量生成其余章节。
