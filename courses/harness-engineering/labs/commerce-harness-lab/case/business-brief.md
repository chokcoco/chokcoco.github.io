# 星桥商城业务简报

## 背景

星桥商城正在准备年中促销。结算页由 React 应用负责价格预览，最终金额由 Java 服务确认。运营后台也会读取促销规则，用于配置活动和处理客诉。

仓库经历过两次促销系统改造。旧代码还在运行，新规则散落在文档、前端和后端。测试主要覆盖正常下单，组合优惠、网络重试和库存超时覆盖不足。

## 需求 A：优惠叠加与价格解释

原始需求：

> 支持会员折扣和优惠券叠加，结算页把优惠写清楚，尽快上线。

业务约定：

- 金额以人民币分为单位保存和计算。
- 会员商品折扣默认九折。
- 订单满 500 元可减 50 元。
- 会员折扣和订单券能否叠加由活动策略决定，不能写死在页面。
- 特价商品不再叠加商品级会员折扣，但仍可能参加订单级满减。
- 结算结果要返回原价、商品优惠、订单优惠、运费优惠和应付金额。

## 需求 B：库存预占与超时释放

原始需求：

> 下单时别超卖，支付超时后把库存还回来，网络重试也不能重复扣库存。

业务约定：

- 创建支付单前预占库存 15 分钟。
- 同一个幂等键重复提交相同请求，应返回同一预占结果。
- 同一个幂等键提交不同商品或数量，应拒绝并记录冲突。
- 支付成功后把预占转为正式占用；取消或超时后释放。
- 重复 claim、cancel 或 release 不得重复改变库存。
- 多仓场景优先选择能一次满足整单的仓库。

## 课程边界

实验仓库不连接真实支付、物流、会员或库存系统。它关注任务契约、仓库知识、工具接口、边界和反馈，不用于演示生产级高并发数据库方案。
