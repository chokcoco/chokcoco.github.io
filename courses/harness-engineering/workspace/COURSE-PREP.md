## Working terminology

| 英文 | 课程中文 | 课程中的含义 |
| --- | --- | --- |
| Harness Engineering | 驭缰工程 | 设计智能体工作环境、意图、能力、控制和反馈的工程方法 |
| agent harness | 智能体运行支架 | 把模型、工具、状态、策略和循环组织起来的系统 |
| agent loop | 智能体循环 | 模型判断、调用工具、观察结果并继续或结束的循环 |
| agent legibility | 智能体可读性 | 智能体能否发现、理解并验证完成任务所需的信息 |
| system of record | 事实源 | 团队和智能体共同依赖、可版本化和可维护的权威信息 |
| invariant | 不变量或硬约束 | 不允许被实现自由破坏的规则 |
| feedback loop | 反馈回路 | 产出、检测、反馈、修正和回归验证组成的循环 |
| eval | 评测 | 用真实样本和标准衡量智能体行为是否符合预期 |
| autonomy level | 自治等级 | 智能体可以独立执行到什么范围，哪里需要人工确认 |
| garbage collection | 持续清理 | 定期发现并清除漂移、坏模式和技术债 |

课程确定使用“驭缰工程”，首次出现时保留 Harness Engineering。
