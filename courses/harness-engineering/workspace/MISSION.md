# Mission: Harness Engineering（驭缰工程）

状态：整课质检阶段 v0.4，13 章与课程实验仓库已生成。

## Why

这门课帮助学习者理解：当 AI 智能体能够读代码、调用工具、运行测试并持续工作时，工程师的价值开始从“亲自完成每一步”转向“把目标、环境、边界和反馈设计清楚”。

课程以 OpenAI 2026 年 2 月 11 日发布的 Harness engineering: leveraging Codex in an agent-first world 为起点。原文描述了一项“仓库中没有人工编写代码”的内部实验，并把团队的主要工作概括为设计环境、明确意图、建立反馈回路，让 Codex 可靠执行。课程不会把这个实验条件宣传成所有团队都必须采用的规则，也不会把 Harness Engineering 简化为“以后工程师都不写代码”。

课程使用“驭缰工程”作为 Harness Engineering 的中文译名，首次出现时保留英文。

## Audience

### 主线受众

- 0 基础或只有轻量编程经验，希望理解 AI 智能体如何参与真实工作的学习者
- 会使用聊天式 AI，但还分不清模型、智能体、工具、工作流和 harness 的学习者

### 进阶受众

- 企业在职研发、测试、架构、DevOps、SRE、研发效能和技术管理人员
- 正在引入 Codex、Claude Code、GitHub Copilot 或其他编码智能体的团队
- 希望把个人提示词经验升级成可复用工程体系的实践者

主线使用共同语言解释概念。面向研发人员的实现细节放入“工程加深”与进阶实践，不要求初学者在第一次阅读时掌握全部代码。

## Learning outcomes

完成课程后，学习者应能：

- 用自己的话解释 Harness Engineering、agent harness 和 agent loop 的区别与关系
- 说明为什么模型能力强不等于智能体能可靠完成真实任务
- 识别一个智能体工作环境中的目标、上下文、工具、权限、状态、反馈和停止条件
- 把模糊需求改写成包含范围、约束、验收标准和失败处理的任务契约
- 设计对智能体可读的知识结构，例如仓库地图、AGENTS.md、架构文档、计划和进度记录
- 区分“写更多提示词”与“把规则编码进测试、lint、schema、hook 和权限策略”
- 建立测试、日志、指标、轨迹、评价量规和人工复核组成的反馈回路
- 解释长时运行任务中的上下文耗尽、状态交接、任务分解、恢复和漂移问题
- 根据任务风险选择人工确认、受限执行、自动修复或持续运行等不同自治等级
- 识别规模化智能体工程中的安全、成本、审计、技术债和组织协作风险
- 为一个小型项目或企业流程设计一份可执行的 harness 改造方案
- 在进阶路径中，把设计落成仓库指令、验证脚本、测试、可观测性和质量报告

## Success evidence

学习完成不能只以“读完章节”为准。至少需要以下证据：

- 概念判断：能从案例中区分模型问题、提示问题、工具问题、环境问题和反馈问题
- Harness 画布：能画出一个任务的输入、执行环境、工具、边界、反馈、状态和人工闸门
- 意图契约：能把一个模糊需求改写成可验证的工作说明
- 仓库改造：能为示例项目补充最小知识地图、指令文件和验证命令
- 失败分析：能根据一次智能体失败记录找到缺失能力，而不是只要求模型“再试一次”
- 毕业实践：提交 harness 设计、运行证据、失败记录、改进循环和风险说明

## Constraints

- 课程使用中文，首次出现的英文术语保留原文并给出准确解释
- 同时照顾初学者与在职研发人员，但不把同一段正文写成两套互相打断的教程
- 核心事实优先使用 OpenAI 原文、官方文档、官方开源仓库和其他一手工程资料
- 明确区分 OpenAI 的具体实现、其他厂商案例和可迁移的一般原则
- 对 2026 年仍在快速变化的产品功能、模型能力和指标标注来源日期
- 不把 OpenAI 内部实验的产能数字直接推广为企业普遍收益
- 课程正文进入生成阶段后必须执行 humanizer-zh、第一阶段确定性质检和第二阶段课程评价量规
- 代码只在帮助理解或完成实践时出现；初学路径允许通过模板和 AI 辅助完成
- 声称可运行的代码、命令和配置必须在对应环境验证
- 总课时控制在 12–15 小时，采用 13 章结构，学习周期不设硬限制
- 理论、实践、案例按 5:3:1 的相对权重规划；抽象理论仍需配通俗案例
- 工具与模型保持厂商中立，案例可涉及 Codex、Claude Code、GLM 和千问
- 代码实践同时提供 JavaScript 与 Java 路径；基础实践兼容 macOS 与 Windows
- 正式交付为 Markdown，供教学平台使用；本仓库同时生成静态 HTML 预览，但不自动对外发布
- 课程计划用于售卖，学生页面不得暴露内部资料、创作过程、来源分级、样章验收或质量报告
- 公共参考文献统一放在章末；内部材料只保留在私有 workspace 的来源映射中
- 主案例采用两步走：先改造已有企业仓库，再从零建设新仓库
- 企业内部材料只用于作者侧分析，不在学生正文、参考文献、链接或页面元数据中披露
- 课程使用一套完全虚构、可公开分发的电商模拟仓库，不使用真实企业代码、客户数据或生产配置
- 前端工程路径采用 React 19.2 与 Vite 8.1；Java 工程路径采用 Java 21、Spring Boot 4.1 与 Maven

## Out of scope

- 不把课程做成单一 Codex 产品操作手册
- 不承诺完全无人监督的软件开发
- 不把提示词技巧等同于 Harness Engineering
- 不要求初学者掌握模型训练、强化学习或复杂分布式系统
- 第一版不实现企业级多租户智能体平台
- 第一版不使用未经授权的内部代码、业务数据或生产凭证

## Human review gate

- 第 1–2 章已通过人工方向确认，可以扩展整课
- 第 3–13 章和实验仓库完成质量闭环后仍需整课人工审核
- 教学平台的 Mermaid、代码高亮和自定义 HTML 支持仍需在最终导入时复核
