## 完课后必须执行的课程专用检查

正式执行第二阶段时，在通用量规之外增加以下证据项：

1. 对第 3–16 章逐章记录 Unicode 有效字符数，以约 10,000 字符作防缩水建议值。低于建议值必须说明知识范围为何较小或进入补充；高于建议值不能自动通过。
2. 按课程 spec 的九步完整性审查判断有效深度：基础概念、语法或 API、运行机制、常用方式、应用边界、易错点和练习是否都有实质解释。重复总结、无关示例和未拆解代码不计为有效内容。
3. 检查学习顺序：任何代码和术语都不得依赖尚未讲解的后续内容；组合示例只能出现在组成部分讲清之后。
4. 检查标题导航：第 3–16 章使用中文序号二级标题和递进编号三级标题，标题准确表达知识点，不套用空泛模板。
5. 抽查全部带输出代码及多步骤代码：预期控制台输出须有相邻注释，关键步骤须有能帮助零基础学习者理解的注释。

6. 给每章记录“完整正文”“需要补充”或“结构性返工”及对应证据；不得只依据字符数判定。

这些要求只登记为后续审校合同。本轮不据此出具正式量规分数或发布结论。
