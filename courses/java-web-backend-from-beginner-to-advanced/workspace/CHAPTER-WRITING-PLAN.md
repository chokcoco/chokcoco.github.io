# Java Web 后端入门到进阶：章节写作规划

状态：第二模块深度返工规划 v0.2。它决定后续正文的组织方式，不把未达到标杆深度的短章节视为最终教学正文。

## 使用规则

- 第 6、21 章是已确认正文范例，保留其“先补足前置、再逐步组合”的节奏。
- 第 1、2 章先按本文件重写，重点是课程地图和环境落地，不使用语言概念课模板。
- 其余章节按“教学任务类型”选择结构。标题是为本章内容服务的工作标题，正式写作时可微调，但不得退回统一模板。
- 各模块学生正文的教学二级标题统一使用中文序号，三级及更深标题使用递进编号；章末固定保留未编号的“参考文献”。第 1、2 章仍按环境落地任务组织，不套用语言概念课模板。第 3–16 章以第 6 章为解释粒度和组织方式参考，采用约 12,000 Unicode 有效字符的防缩水基准，不复制第 6 章的精确长度。低于基准需要说明或补充，高于基准仍须通过九步完整性审查。
- 带输出代码写明预期控制台结果，多步骤代码在关键动作处补充注释。注释解释学习障碍，不机械逐行翻译。

## 第 1 章：Java Web 后端学习地图与开发环境

教学任务：建立方向感和课程预期；不要求安装工具或记忆框架细节。

建议正文组织：

1. 前端工程师进入后端后多了哪些职责
2. 一次页面操作怎样到达 Java 服务
3. Java、Spring Boot、数据库、Redis、消息队列和 Docker 分别放在什么位置
4. 为什么课程以 JDK 17 为主线，以及 JDK 21+ 内容会如何出现
5. 36 章的学习路线与每个模块的完成标志
6. 开始第 2 章前需要准备的权限、终端和工具

验收：学习者能画出“浏览器 → HTTP → Java 服务 → 数据库”的简图，能说出 Java 语言、Spring Boot 与数据库的区别，并知道下一章结束时要得到什么结果。

## 第 2 章：JDK 17、IDE 与 Maven 环境核验

教学任务：在一台新电脑上建立可复现环境，并留下可以排错的版本证据。

建议正文组织：

1. 先看完成后的终端、IDE 与 Maven 核验结果
2. 分清 JDK、JVM、`java` 与 `javac`，选择官方 JDK 17 下载渠道与正确 CPU 架构
3. 分别完成 macOS 与 Windows 的 JDK 安装、终端/PowerShell 版本核验
4. 在终端创建、编译和运行最小程序
5. 解释 IDE 的作用，选择并在 macOS/Windows 安装 IntelliJ IDEA；说明 VS Code、Eclipse 的替代边界
6. 配置 IDE 的 Project SDK，并运行同一个程序
7. 分别安装 macOS 与 Windows 的 Maven，用最小 POM 完成一次构建
8. 按 PATH、`JAVA_HOME`、IDE SDK、Maven Runner 的顺序排查版本不一致
9. 保存环境核验记录

验收：macOS 或 Windows 上的终端、IDE、Maven 都使用 Java 17；学习者可以解释其中任意一个不一致时先检查什么。
