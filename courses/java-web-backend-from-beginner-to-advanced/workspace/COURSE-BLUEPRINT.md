# Java Web 后端入门到进阶：课程蓝图

状态：36 章大纲 v0.3，按 Java 零基础和不限单章时长重新校准。

## Course spine

```text
开发环境与最小程序
  → Java 基本语法、方法、类型与引用
  → 面向对象、异常、泛型、集合与标准库
  → HTTP、Spring Boot、REST 与错误处理
  → SQL、数据访问、事务、测试与 Docker
  → Security/JWT、Redis、Kafka、并发与 JVM
```

核心路线：1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 10 → 11 → 12 → 13 → 14 → 17 → 18 → 20 → 21 → 23 → 24 → 25 → 26 → 27 → 29 → 30 → 31 → 32 → 33 → 34 → 35 → 36。

每章都从学习者尚未掌握本章前置的角度开始，但正文结构不统一：语言与概念章节按递进解释展开，环境、框架、数据、部署、安全和诊断章节按操作链路、可观察结果、失败模式和验收证据展开。逐章结构见 `CHAPTER-WRITING-PLAN.md`。表中的“展开重点”表示应讲透的内容，不代表其他项可以略讲。

第 3–16 章采用约 10,000 Unicode 有效字符的防缩水建议值，并逐章执行九步完整性审查。第 6 章提供解释粒度和长文导航参考，不提供精确字符硬门槛。二级标题使用中文序号，三级标题使用递进编号；内容状态由知识范围、前置、六层深度、示例、练习、有效篇幅和跨章关系共同决定。

## Dependency notes

- 第 3 章先讲程序的编译与运行，第 4 章必须提前讲方法定义、形参、实参、返回值和调用；第 6 章才能讨论值传递。
- 第 6 章只用 String、基本类型和本章已经补足的最小方法语法做核心练习。自定义类、record、集合分别留给第 8、9、13 章。
- 第 17 章先讲 HTTP，第 18 章再讲 Spring Boot；第 20 章完成分层后，第 21 章才引入请求 DTO、校验和全局异常。
- 第 21 章使用 curl 做基础验证。MockMvc 和测试分层在第 27 章系统讲解，不提前把测试 DSL 塞进正文。
- 第 23 章先讲 SQL，第 24、25 章再使用 MyBatis-Plus 与 JPA；第 26 章建立事务边界。
- 第 30–35 章每章都先增加一段基础模型，不以企业故障或复杂配置直接开场。

## Practice spine

练习先独立、后组合：

- 语言阶段：变量计算、字符串、方法调用、自定义类、集合处理
- Web 阶段：最小 GET/POST、参数绑定、校验、错误响应和分层
- 数据阶段：单表查询、分页、写入、事务回滚和测试
- 进阶阶段：最小登录、缓存、消息、并发和 JVM 观察

订单服务只用于基础能力形成后的迁移练习。第 29 章形成可运行闭环，第 36 章分析故障包；中间章节不要求维护大型项目。
