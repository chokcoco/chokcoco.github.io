# Mission: Java Web 后端入门到进阶

状态：第二模块深度返工 v0.5，教学起点为 Java 零基础。

## Why

这门课帮助正在转向全栈开发的前端工程师，从零建立 Java 与 Java Web 后端能力。课程先把语言概念、语法和基本用法讲清楚，再逐步进入 Spring Boot、数据访问、安全、缓存、消息与运行诊断。

学习者完成课程后，应能读懂一个常见 Java Web 服务，独立完成范围明确的前后端需求，并知道何时需要数据库、安全或运维同事协助。

## Audience

### 主线受众

- 没有 Java 基础、正在从前端转向全栈的工程师
- 能使用计算机和开发工具，但尚未系统学习 Java 语法与后端开发的学习者

### 次级受众

- Java 知识零散，希望重新补齐语言基础和 Web 后端链路的初级开发者
- 需要读懂团队 Java 服务并完成小型需求的测试、客户端或研发效能工程师

## Assumed prerequisites

- 会进行基本的文件操作，能够跟随步骤使用 IDE 和终端
- 有 JavaScript、TypeScript 或其他语言经验会学得更快，但不是正文省略基础知识的理由
- 不预设学习者掌握 Java 的变量、方法、类、注解、异常、集合、HTTP、SQL、Linux、容器、缓存或消息队列
- 一章使用某段语法前，该语法必须已经在前文讲过；无法避免时，先在本章补足最小前置知识

## Learning outcomes

完成课程后，学习者应能按下面的顺序建立能力：

1. 读写 Java 程序的基本结构，理解标识符、变量、字面量、类型、运算符、分支、循环、数组和方法的基础语法。
2. 解释基本类型、引用类型、对象、值传递、相等性和空值的含义，并用小程序验证自己的判断。
3. 使用类、接口、泛型、集合、异常、Lambda、Stream、文件和日期时间 API 编写清楚的 Java 代码。
4. 安装和核验 JDK 17，使用 Maven 编译、测试、打包和运行工程，并识别 JDK 后续版本中影响实际开发的差异。
5. 理解 HTTP 与 Spring Boot 的基本工作方式，建立分层 REST API，处理 JSON、参数校验和全局异常。
6. 使用 SQL、MyBatis-Plus 和 JPA 完成基础数据访问，理解分页、查询和事务边界。
7. 为 Service、Controller 和数据访问边界编写有价值的自动化测试，并用 Docker 运行应用及其依赖。
8. 在掌握上述基础后，理解 Spring Security、JWT、Redis、Kafka、线程池和 JVM 的基本机制、常见错误与应用边界。

## Success evidence

- 基础语法：能从空文件写出可编译程序，逐行解释变量、表达式、控制流和方法调用
- 语言原理：能画出变量与对象的关系，解释值传递、`==`、`equals`、`hashCode` 和 `null`
- 标准库：能根据顺序、去重、查找、缺失值和数据转换需求选择合适 API
- Web：能从 HTTP 请求解释到 Controller 参数绑定、校验、业务处理和错误响应
- 数据：能写出必要 SQL，并用 MyBatis-Plus 或 JPA 完成数据访问与事务内修改
- 安全与中间件：能先说明基础概念，再解释认证、缓存、消息和并发方案的失败边界
- 工程：能运行测试和构建，读取日志与堆栈，并用 Docker 启动完整示例

## Constraints

- 课程名称为「Java Web 后端入门到进阶」，保持 36 章和四个模块；“环境介绍与准备”只保留第 1–2 章，后续时间用于语言与后端基础
- 不再限制单章时长，也不以原定 36 小时总时长压缩知识点；章节以“讲清楚并能验证”为完成标准
- 以 JDK 17 为可运行主线，后续版本只讲影响代码阅读、迁移或运行决策的重要差异
- 主框架采用 Spring Boot 3.x；数据访问同时介绍 MyBatis-Plus 与 JPA，数据库同时覆盖 MySQL 与 PostgreSQL
- Kafka 是消息实作主线；RabbitMQ 与 RocketMQ用于模型与场景对照
- 不建设大型项目；先使用短小示例解释概念，掌握基础后再进入局部业务场景
- 所有章节标题使用直叙句，不使用疑问句
- 不把同一写作模板强加给 36 章：环境章节以落地与核验为主，概念章节以递进解释为主，框架、数据、部署、安全和诊断章节以任务链路、操作、证据和排错为主
- 环境、IDE、构建、容器、路径和命令类内容同时支持 macOS 与 Windows；首次使用的工具必须给出零基础可完成的官方安装、配置、核验和排错路径
- 学习者必须访问才能完成当前操作的官方地址直接放在相应步骤中；课程对零基础保持解释充分，但删除作者内心独白、重复提醒和对读者能力的揣测
- 中文学生内容执行 humanizer-zh、确定性质检、课程评价量规和浏览器体验检查
- 可运行代码必须在 JDK 17 环境实际编译、测试或启动
- 第 3–16 章以第 6 章为解释粒度参考，以约 10,000 Unicode 有效字符作为防缩水基准；章节最终是否完整由知识范围、零基础前置、六层深度、示例、学习闭环、篇幅有效性和跨章关系等九步审查共同判断
- 第二模块长章使用中文序号二级标题与递进编号三级标题；代码的可观察输出和多步骤关键动作在代码块内就近注释

## Out of scope

- 完整微服务体系、服务注册发现、网关、分布式配置和全链路治理
- OAuth2/OIDC 授权服务器建设、SSO、多租户权限与复杂 ACL
- Redis Cluster、消息集群运维、跨机房容灾与性能压测
- 深入 JIT 编译、GC 参数调优、字节码插桩和 JVM 源码
- Kubernetes、云厂商部署、CI/CD 平台建设与生产运维值守
- 在同一个业务工程中混用 MyBatis-Plus 与 JPA
- 算法竞赛和面试题库

## Human review gate

用户确认第 6 章的内容与写法可作为语言与概念章节的基准。环境、框架、数据、部署、安全与诊断章节应继承其“前置清楚、循序渐进、代码不越级”的原则，但采用适合自身教学任务的结构。
