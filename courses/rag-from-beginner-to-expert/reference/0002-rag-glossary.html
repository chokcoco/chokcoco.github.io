RAG 术语速查表

这份材料用于复习和快速查阅。建议打印或和 lesson 并排打开。

RAG

Retrieval-Augmented Generation，检索增强生成。回答前先从外部知识源检索资料，再基于资料生成答案。

LLM

Large Language Model，大语言模型。它擅长生成语言，但不会自动知道你的企业私有资料。

参数化知识

模型参数中固化的知识，更新成本高，不适合频繁变化的企业资料。

非参数化知识

外部知识库中的资料，可更新、可替换、可审计，是 RAG 的核心来源。

embedding

向量嵌入。把文本、图片或其他内容转成数字向量，用于计算相似度。

token

模型处理文本的基本单位，可近似理解为字、词或词片段。

chunk

文本块。长文档被切分后的最小检索单元。

overlap

分块重叠。相邻 chunk 共享一部分内容，用来减少边界信息丢失。

向量数据库

存储向量和元数据，并支持相似度检索的数据库或索引系统。

稀疏检索

关键词或倒排索引检索，适合专有名词、错误码、订单号等精确匹配。

密集检索

基于 embedding 的语义检索，适合同义问法和自然语言表达。

混合检索

结合稀疏检索和密集检索，兼顾精确匹配和语义匹配。

top_k

检索时返回前 k 个候选结果。k 太小可能漏召回，k 太大可能带来噪声。

rerank

重排。对初次召回的候选结果进行二次排序，提高最终上下文质量。

查询构建

把自然语言问题转换成结构化查询、过滤器或路由指令。

Text2SQL

把自然语言问题转换成 SQL 查询，适合连接关系型数据库。

查询重写

把用户原始问题改写、拆解或扩展成更适合检索的表达。

多跳检索

先找到一批资料，再基于中间结果继续追问或追查下一批资料的检索方式。

上下文包

最终交给 LLM 的材料组合，包含问题、检索片段、规则、引用和输出格式。

元数据

Metadata，描述资料的数据，例如来源、版本、业务域、权限、更新时间和负责人。

知识物料

为 RAG 检索和引用而整理过的知识单元，通常包含正文、来源、版本、业务域、权限和负责人。

数据血缘

记录资料从哪里来、经过哪些处理、最终进入哪个索引的追踪信息。

GIGO

Garbage In, Garbage Out，输入垃圾，输出也会变差。RAG 中尤其指低质量资料会直接污染答案。

权限过滤

在检索阶段按用户身份、角色、组织或数据密级过滤资料，防止越权召回。

召回

检索阶段把可能相关的资料找回来的动作。召回不到正确资料，后续生成很难正确。

可观测性

记录一次 RAG 请求的检索、重排、生成、引用、成本等中间过程。

评测集

固定问题、标准答案、期望引用和评分规则的集合，用于判断系统是否变好。

GraphRAG

基于知识图谱的 RAG。将实体和关系组织成图，用于跨文档关系问题。

多模态 RAG

不仅检索文本，也检索图片、表格、视频帧等多模态内容。
