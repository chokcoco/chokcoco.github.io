# RAG 从入门到精通：课程创建前准备清单

这份文件是课程正式生成前的任务书。你可以直接在每个 `待确认` 后面填写答案；我会根据你的补充再开始创建具体 lesson、reference 和 assets。

## 1. 课程定位

- 课程名称：RAG 从入门到精通
- 一句话承诺：让 0 基础学习者知道 RAG 为什么重要，并能在引导下搭建一个企业知识库原型。
- 目标学习者：没有编程经验、听说过 AI 或 ChatGPT，但不了解 AI 工程实现的人。
- 课程深度：从概念理解到低代码/无代码企业知识库方案，不默认进入代码实现。
- 待确认：学习者更像企业老板、产品经理、运营、知识管理人员、培训讲师，还是准备转 AI 产品/工程岗位的人？ 
  - 回答：学习者面向有简单的编程经验（或者完全 0 基础的），譬如在校大学生，可能是计算机科班也可能是想转行编程的大学生，他们知道 AI 是目前比较火热的行业，希望通过一系列课程，了解 AI 相关的各个含义和概念，最终能够熟练使用各种 AI 工具和模型，找到企业相关工作（譬如 AI Agent 工程师、全栈开发工程师），或者是能够从不懂 AI 到慢慢转向独立AI开发工程师；

## 2. 学习目标

学习者完成课程后，应能做到：

- 解释 RAG 的业务价值：降低幻觉、连接企业私有知识、支持资料更新、给出来源。
- 识别 RAG 的基本组件：资料源、切分、embedding、向量库、检索器、重排器、生成模型、引用、评测。
- 使用一个可视化类比理解核心流程：像“开卷考试”，AI 先查资料再回答。
- 判断适用场景：客服、HR、销售支持、政策问答、产品文档助手、内部制度查询。
- 搭建知识库原型：上传资料、配置问答、提问、检查答案是否引用资料。
- 说出企业级落地约束：权限、数据更新、文档质量、答案评测、成本、日志、安全。

待确认：你希望最终“搭建企业级知识库”的实践使用哪类工具？ 回答：因为面向的是学生，最终的知识库尽量选择他们比较容易操作的方式，我希望有两个版本，一个是先无代码，最后附一个可选代码扩展的选项，但是同时也提供类似「Dify、Coze、扣子、FastGPT、AnythingLLM 等低代码工具」创建企业级知识库的指引。

- 无代码知识库平台
- OpenAI file search / vector store 思路
- Dify、Coze、扣子、FastGPT、AnythingLLM 等低代码工具
- LangChain / LlamaIndex 代码演示
- 先无代码，最后附一个可选代码扩展

## 4. 推荐课程结构草案

第一版建议 6 个模块，每个模块 3-5 节短 lesson。

### Module 1: 为什么需要 RAG

- AI 为什么会答错
- 企业知识为什么不在模型脑子里
- RAG 的一句话解释：先检索，再生成
- RAG、搜索、微调、prompt 的区别

### Module 2: RAG 的核心流程

- 文档从哪里来
- 文档为什么要切分
- embedding 和向量的直觉
- 检索器如何找相关内容
- 生成模型如何基于资料回答

### Module 3: 从玩具知识库到可用知识库

- 什么是好资料
- chunk 大小和重叠的直觉
- 关键词检索、向量检索、混合检索
- reranking 为什么重要
- 引用和可追溯性

### Module 4: RAG 的实际应用场景

- 客服知识库
- HR/制度问答
- 产品文档助手
- 销售和售前支持
- 企业内部知识助手

### Module 5: 搭建企业级知识库

- 需求定义和资料盘点
- 权限和数据安全
- 更新流程和版本管理
- 问答质量评测
- 成本、速度和用户体验

### Module 6: 毕业项目

- 选择一个企业场景
- 准备 10-20 份资料
- 搭建知识库原型
- 设计 20 个测试问题
- 评估答案准确性、引用质量和拒答能力

待确认：课程总时长希望是 2 小时速成、1 天工作坊、7 天训练营，还是 4 周系统课？
回答：RAG 不是一个特别复杂的知识，我希望通过 2-4 小时就能较好的掌握 RAG，通过 1-2 天的实践可以借助一些平台快速搭建出一个知识库，因此这里我希望整体的课程理论部分是2-4小时，如果有涉及实践部分，控制在1-2天（10小时左右）；

## 6. 质量标准

每节 lesson 应满足：

- 只解决一个明确问题。
- 先用生活类比建立直觉，再给术语。
- 至少包含一个 retrieval practice 小练习。
- 至少包含一个常见误区。
- 链接一个主要可信来源。
- 最后给出一个“你现在能做什么”的小胜利。

整门课应满足：

- 概念解释前后一致，术语以 `GLOSSARY.md` 为准。
- 练习从识别、排序、判断，逐步过渡到真实知识库设计。
- 不把“能跑起来”误当成“能上线”。
- 最终毕业项目能检验学习者是否真的理解 RAG 的价值和限制。
